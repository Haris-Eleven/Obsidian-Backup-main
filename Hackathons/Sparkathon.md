5. Architecture
Layers:

Data Sources: Real-time inventory data, sales trends, expiry dates, in-store sensors, and external data on donations.

Ingestion Layer: Collects stock and spoilage data through APIs and POS systems.

Processing Layer: Runs AI models (e.g., LLaMA 3, YOLO for visual spoilage detection) to identify waste drivers and generate insights.

Backend Services: Flask APIs, Celery tasks for periodic model runs, PostgreSQL to store trends and recommendations.

Frontend: React dashboards for store associates and managers, showing waste hotspots, suggested actions, and impact metrics.

Security: OAuth2 and JWT for secure associate logins and role-based access. ^dU7xt1tU
